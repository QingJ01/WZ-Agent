# GUI 使用说明

启动命令：

```powershell
python GUI.py
```

---

## 设备连接与运行

| 控件 | 说明 |
|------|------|
| 设备模式 | `自动` / `真机 ADB` / `MuMu ADB` |
| ADB 路径 | 选择 `adb.exe` 的位置 |
| ADB 设备 | 刷新后自动选择，也可手动填写 serial |
| AI 自动操作 | 关闭时只检测和记录，不向设备发送战斗操作 |
| 小地图预览 | 在 GUI 中显示最新小地图裁剪图 |
| EVE 调试窗口 | OpenCV 调试窗口，普通使用时关闭 |

> 建议先关闭 AI 自动操作跑一局，确认画面检测和记录正常后再开启。

---

## 示范数据采集

用于录制人工操作数据，供策略模型训练使用。

| 配置项 | 说明 |
|--------|------|
| 输入源 | 自动选择：真机模式用 ADB 触摸，MuMu 模式用 Windows 键盘 |
| 示范目录 | 默认 `logs/human_demos` |
| 游戏画面分辨率 | 默认 `2400x1080`（横屏游戏坐标平面） |
| 原始触摸分辨率 | 真机可自动检测，检测失败时手动填写（如 `10800x24000`） |
| 触摸坐标变换 | 默认 `identity` |

录制数据会附带坐标元数据，训练前自动检查坐标是否越界。

---

## 策略模型训练

GUI 内置训练入口，数据类型会自动切换目录和输出：

| 数据类型 | 输入目录 | 输出模型 |
|----------|----------|----------|
| 人工示范 | `logs/human_demos` | `models/human_policy.pt` |
| 自训练记录 | `logs/decision_records` | `models/self_policy.pt` |

训练完成后 GUI 显示训练报告：

- 样本数与动作覆盖
- 训练/验证准确率
- 动作分布 Top5
- 缺失动作与样本过少警告
- 模型是否可启用

> 如果报告显示「模型不可启用」，请勿勾选该模型运行。需要继续采集数据或清理污染数据。

---

## 模型与推理配置

| 配置项 | 说明 |
|--------|------|
| YOLO 检测模型 | 三个 YOLO 权重路径 |
| 策略模型路径 | `self_policy.pt` 或 `human_policy.pt` |
| 接管置信度阈值 | 模型输出置信度高于此值才接管规则决策 |

启用策略模型前，GUI 会检查同名训练报告。没有报告的模型不会被强制拒绝，但不建议使用未经验证的模型。

---

## 日志

GUI 日志区合并显示：

- ADB 刷新结果
- 运行进程 stdout/stderr
- 训练进程输出
- 训练报告摘要
- 启动配置错误

**乱码问题**：GUI 默认为子进程设置 `PYTHONIOENCODING=utf-8:replace` 和 `PYTHONUTF8=1`。如果命令行启动出现乱码，手动设置这两个环境变量即可。
